# Contributing Guidelines (C-SET)

## Scope
This repository hosts modular, reproducible pipelines for transplant exposome research. No raw or identifiable data should be committed.

## Branching
- Use `main` for stable code.
- Create feature branches using: `feat/<topic>`, `fix/<topic>`, or `docs/<topic>`.

## Pull Requests
- Keep PRs focused and small when possible.
- Include a short summary of changes and rationale.
- Add or update documentation when introducing new modules.

## Code Standards
- Prefer modular functions over monolithic scripts.
- Include clear parameter definitions and default values.
- Ensure workflows are reproducible from raw inputs (within secure environments).

## Security and Data Protection
- Do not upload raw data, extracts, or any PHI/PII.
- Use only synthetic examples or schema-level documentation.

## Issues
Use GitHub Issues for:
- Feature requests
- Bug reports
- Documentation improvements
